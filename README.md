# site-500-pages

这是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定网站或域名，仅用于存放和展示一系列独立的网页文件。

## 项目简介

本项目旨在提供一个集中存放 HTML 页面的空间，方便对多个独立页面进行版本管理与分享。每个页面均为自包含的 HTML 文件，不依赖外部数据库或后端服务，可直接在浏览器中打开查看。

## 目录说明

```
site-500-pages/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面引用的公共资源（如 CSS、JS、图片等）
└── index.html      # 可选的首页或导航页
```

- `pages/`：核心目录，所有独立的 HTML 页面均存放于此。每个文件以有意义的文件名命名，便于识别。
- `assets/`：存放页面间共享的样式表、脚本、图片等静态资源。
- `index.html`：可选文件，可作为页面索引或简单导航。

## 页面归档说明

- 每个 HTML 页面均为独立、自包含的文档，可单独访问。
- 页面间无强制依赖关系，但可引用 `assets/` 目录下的公共资源。
- 归档的页面内容不涉及具体商业、营销或推广信息，仅作为技术展示或资料留存。
- 页面文件名建议使用英文小写字母、数字及连字符，避免特殊字符。

## 使用方式

1. 克隆或下载本仓库。
2. 在浏览器中直接打开 `pages/` 目录下的任意 HTML 文件即可查看。
3. 若启用 GitHub Pages，可通过仓库设置中的 Pages 功能直接访问页面内容。

## 维护说明

- 新增页面：在 `pages/` 目录下添加新的 HTML 文件，确保文件结构完整、引用正确。
- 更新页面：直接修改对应 HTML 文件，建议在提交时附上简要的变更说明。
- 删除页面：移除不再需要的文件，并检查是否存在相关引用。
- 资源管理：公共资源请统一放置在 `assets/` 目录下，避免重复引入。
